# Accessibilité, confort et sécurité sociale

## Objectif

Tomate ! doit rester drôle sans transformer la gêne de jeu en gêne personnelle. Le jeu vise des groupes d'amis, des communautés en ligne et potentiellement des contextes de streaming ou d'atelier.

## Accessibilité de lecture

- taille de texte réglable ;
- largeur de colonne limitée ;
- interlignage réglable ;
- police sans empattement disponible ;
- mode contraste renforcé ;
- désactivation des tremblements et rotations ;
- réduction des animations ;
- perturbations qui ne reposent pas uniquement sur la couleur ;
- raccourcis clavier pour confirmer une réplique et nettoyer l'écran ;
- compatibilité tactile ;
- annonces adaptées aux lecteurs d'écran pour les changements essentiels.

## Paramètres individuels

Chaque joueur peut choisir localement :

- intensité visuelle maximale ;
- volume des bruitages ;
- réduction des flashs ;
- désactivation des secousses ;
- affichage d'un fond opaque sous le texte ;
- durée supplémentaire de lecture ;
- mode sans chronomètre visible ;
- absence de ciblage direct ;
- mode « je préfère lire » sans improvisation obligatoire.

Ces réglages ne doivent pas être présentés comme un avantage compétitif.

## Budget de gêne

Le serveur calcule un budget par cible.

Exemple de catégories :

- gêne visuelle ;
- gêne sonore ;
- contrainte d'interprétation ;
- altération du texte ;
- pression temporelle.

Règles initiales :

- une seule gêne majeure par catégorie ;
- deux gênes majeures maximum simultanément ;
- aucune occultation totale de la réplique active ;
- une fenêtre de récupération après un incident fort ;
- durée maximale configurable ;
- coût croissant lorsque la même réaction est répétée.

## Anti-acharnement

- impossibilité de cibler continuellement le même joueur ;
- délai individuel après une réaction ciblée ;
- coût augmenté pour les répétitions ;
- bonus de défense ou d'aide après plusieurs incidents ;
- réactions collectives soumises à un seuil de vote ;
- option « public non ciblé » où le serveur choisit la cible ;
- bouton hôte pour suspendre les réactions ;
- exclusion et blocage depuis le salon.

Le journal interne doit permettre de distinguer une forte activité collective d'un comportement répété provenant d'une seule personne.

## Consentement et thèmes

Avant une session, l'hôte peut désactiver des catégories :

- romance ;
- horreur ;
- violence fictive ;
- humiliation ;
- insultes jouées ;
- politique ;
- religion ;
- contenu adulte ;
- bruitages soudains.

Le contenu généré ou communautaire doit déclarer ses catégories. Un joueur peut appliquer ses propres exclusions sans avoir à les justifier.

## Audio et enregistrement

- aucun enregistrement par défaut ;
- consentement explicite de tous les participants avant capture ;
- indicateur permanent lorsqu'un enregistrement est actif ;
- possibilité de jouer sans microphone intégré ;
- suppression simple des captures ;
- durée de conservation annoncée ;
- séparation entre métriques de jeu et contenu vocal ;
- aucune analyse émotionnelle cachée.

## Streaming

Le mode streaming devra prévoir :

- délai ou modération des réactions du chat ;
- liste blanche de réactions ;
- filtrage des noms et textes personnalisés ;
- possibilité pour le streamer de masquer le code du salon ;
- contrôle d'urgence ;
- journal de modération ;
- limitation des nouveaux comptes ou spectateurs anonymes.

## Photosensibilité et mouvements

- pas de flash rapide par défaut ;
- mode sans flash ;
- animations limitées en fréquence ;
- effets de secousse désactivables ;
- aucun effet critique transmis uniquement par mouvement ;
- avertissement clair pour les packs spéciaux.

## Échec doux

Le jeu ne doit pas humilier un joueur bloqué.

Outils de récupération :

- souffleur ;
- reformulation autorisée ;
- passage à la réplique suivante ;
- pause courte contrôlée par l'hôte ;
- réduction automatique du chaos ;
- conclusion de secours ;
- possibilité de céder le rôle sans pénalité publique.

## Modération du contenu communautaire

À prévoir avant l'ouverture d'un éditeur public :

- signalement ;
- statut privé, non listé ou public ;
- validation automatique de structure ;
- filtres de vocabulaire configurables ;
- catégories d'âge et de contenu ;
- historique des versions ;
- retrait rapide ;
- blocage d'un auteur ;
- règles spécifiques pour les textes importés et les droits d'auteur.

## Checklist minimale avant un test public

- [ ] mode contraste renforcé ;
- [ ] réduction des animations ;
- [ ] contrôle du volume ;
- [ ] suspension des réactions par l'hôte ;
- [ ] limite de gêne simultanée ;
- [ ] exclusion d'un joueur du salon ;
- [ ] aucun enregistrement implicite ;
- [ ] avertissements de contenu visibles ;
- [ ] possibilité de terminer une manche sans improviser.
