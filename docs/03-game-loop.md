# Boucle de partie

## 1. Salon

Les joueurs rejoignent avec un code court. L'hôte choisit :

- le mode ;
- la durée ;
- les rôles humains ou automatisés ;
- le niveau de chaos ;
- le type de contenu ;
- les options d'accessibilité.

## 2. Distribution

Le jeu attribue les rôles selon :

- le mode ;
- les préférences ;
- les rôles déjà joués ;
- le nombre de participants ;
- l'expérience des joueurs.

## 3. Thème

Un thème est sélectionné ou voté.

Exemple :

> Une tragédie romantique dans une station-service hantée.

Le système associe ensuite personnages, objectifs, contraintes et ressources techniques.

## 4. Préparation

Durée cible : 20 à 90 secondes selon le mode.

Chaque rôle prépare uniquement sa partie. Les informations partagées doivent rester limitées pour préserver l'asymétrie.

## 5. Représentation

Durée cible : 90 secondes à 5 minutes.

La scène continue malgré les erreurs. Les événements critiques deviennent des invitations à improviser plutôt que des états d'échec.

## 6. Montée du chaos

Le public gagne progressivement de nouvelles réactions. Le système adapte leur puissance en fonction de :

- la compréhension actuelle ;
- le rythme ;
- la lisibilité du texte ;
- le nombre d'incidents récents ;
- la performance des joueurs.

## 7. Finale

Un signal annonce la dernière phase. Les joueurs doivent conclure l'histoire, même si elle s'est éloignée du scénario initial.

## 8. Verdict

Le verdict distingue plusieurs axes :

- histoire compréhensible ;
- interprétation ;
- synchronisation ;
- respect du thème ;
- récupération ;
- divertissement ;
- chaos maîtrisé.

## 9. Souvenir de partie

Le jeu génère un résumé partageable :

- titre de la pièce ;
- meilleure improvisation ;
- plus gros incident ;
- phrase finale ;
- critique du public ;
- capture ou mini-affiche éventuelle.
